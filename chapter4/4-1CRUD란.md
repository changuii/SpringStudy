- CRUD 실습

## 서비스의 자원

뉴스기사  
- 제목, 본문  
- 신문사  
- 기자
- 뉴스기사의 분류

식당  
- 메뉴
- 요리사

자원과 자원 사이의 관계들을 잘 생각해볼 것  

웹 서비스  
- 사용자가 원하는 자원의 관리
- 배달의 민족 : 식당, 식당에 대한 정보(분류, 요리, 위치 등등..)
- 자원 제공, 제작, 갱신 등
- (배달 주문을 한다.)
- 부수적인 기능은 서비스별로 다르게 만들어진다.

## Create, Read, Update, Delete

예시. 주소록
- Create : 사용자의 연락처를 생성
- Read : 연락처 정보를 조회, 읽어들인다.
- Update : 번호가 바뀌거나 그런 상황에 갱신해준다.
- Delete : 문제가 생긴 정보거나 필요없다면 삭제한다.

Create : 생성  
Read : 조회  
Update : 갱신  
Delete : 삭제  
이 4가지의 기능은 모두 할 수 있어야지 서비스로서 자원을 활용할 수 있다.  

CRUDL, BREAD, DABE 등 다양한 용어를 사용하지만 근본은 CRUD이다.
